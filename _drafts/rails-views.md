# Rails Views

## Templates

Templates in Rails are used to build views.

* Combine variables with a `.html.erb` file to build a HTML view
* Combine variables with a `.xml.builder` file to build an XML document
* Combine variables with a `.json.jbuilder` file to build JSON

Quite often, Templates will be named after a controller action.

```
app/views/users/index.html.erb
app/views/users/new.html.erb
app/views/users/edit.html.erb
...
```

A templates named after a **controller action** will cause that file to be
rendered by default.

**Example**: The `UsersController#index` method renders
the `app/views/users/index.html.erb` template by default.

## Partial Templates (or Partials)

Use partials to extract shared code between templates. The canonical example
of this is the `_form.html.erb` file, that share code between the `new` and
`edit` templates.

## Layouts

Layouts are used to render content around the output of Rails controller actions.
For example, controllers that inherit from `ApplicationController`, will have
their content wrapped by the `views/layouts/application.html.erb` file. The
`yield` statement indicates where the content generated by the controller will
end up.
